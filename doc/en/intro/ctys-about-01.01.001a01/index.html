The Blueprint

The UnifiedSessionsManager provides user space tools for local and remote on-demand operations.

It scales from a single core CPU of a desktop PC up to huge multi-core multi-CPU server farms.

It relies on proven state-of-the-art components by providing glueware for integration and execution.

It covers a considerable share of available products and provides for easy command-line-level integration of almost any additional.

Supported Systems


  • The current version supports:
    • Hypervisors and Emulators:
      KVM, QEMU, VirtualBox, VMware(Workstation/Server/Player-V1+V2), and Xen
    • Operations Systems
      Linux(CentOS, debian, Enterprise Linux Server-5, Fedora, Mandriva, MeeGo, OpenSUSE, ScientificLinux, SuSE, Ubuntu),
      FreeBSD, OpenBSD, OpenSolaris, Solaris
      VMware-ESX-Login(CLI), XenServer-Login(CLI,X11,VNC)
    • Graphical Userinterfaces:
      Gnome, KDE, X11, Xfce, FVWM
      TightVN, TigerVNC, RealVNC, rdesktop/RDP, Xterm, Gnome-Terminal, CLI, and Emacs
      Native-Consoles of Hypervisors.

  • The next versions are going to add:
    • OpenVZ, VMware(ESX/ESXi), and XenServer
    • NetBSD, (ILLUMOS - when available)
    • QNX, and uCLinux - initially for X86 and ARM architectures, RTEMS.

Your personal CloudComposer


  • The UnifiedSessionsManager is designed as a general purpose toolset for distributed services and systems.

  • Therefore it perfectly fits to the seamless personal usage and management of private and public Cloud Services. The main areas of application are Systemadministration, Software-Development, Simulation, and Testautomation.

  • The provided facilities add extended features for virtual services, such as task distribution, virtual nameservices, and inventory management.

  • The inherent security concept based on OpenSSH provides reliable applicability for sessions to local and remote sites.

Generic View to Services...



  • The detailed analysis of the common tasks used within virtualized environments lead to sessions management facilities by a common generic approach.

  • Current concepts comprise flat virtual appliances - v-apps - based on the following operational paradigms:

    • Desktop Virtualization
      ...is an inherently interactive user session requiring a local user interface.

    • Client Virtualization
      ...is basically the same as desktop virtualization.

    • Server Virtualization
      ...could be seen as a non-interactive service, providing task oriented machine-access and an interactive console access for provisioning and maintenance.

=> A Common View



  • The UnifiedSessionsManager does not distinguish between types of virtualization. The remaining distinction is a logical 3-layered approach of serving categories:

    • HOSTs: Native access to the contained Operating Systems.
    • VMs: Virtual Machines - Acces to Hypervisors.
    • PMs: Physical Machines - Management of physical machines.

  • The logical categorization provides the definition of dependencies for automated operations and nested containment hierarchies.

  • This founds the basic conceptual building block of v-components .

The big Picture

The resulting overall-architecture depicts an idea for the full scope of features supported by the UnifiedSessionsManager.


The primary vision behind the scenes could be summed up as: "A generic network infrastructure of location independent dynamically relocatable services."


The concept of v-stack provides dynamic scalability and vendor independent provisioning with standard management tools.


The design is targeted for administration, development, simulation, and test of IT infrastructures and distributed applications. The runtime environment scales from desktop PCs to to data centers.

How does this fit to Clouds and Services?

What is defined now?



  • The current definition provided by the UnifiedSessionsManager is a first toolset for application of SOA and SaaS as a personal environment.

  • Additionally stacked VMs - v-stacks - are defined as a vertical design concept, providing a generic component interface of virtualized services - v-components - as a common toolset.

  • The introduction of CPUs with huge numbers of cores will enable the concept of v-components as a vertical logical design and administrations concept with a flat physical execution environment, where the tasks are actually dispatched and executed on an array of independent CPUs and cores, thus without performance losses.
